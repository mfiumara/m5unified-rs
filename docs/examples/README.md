# Example coverage

This directory tracks the upstream M5Unified example inventory and the Rust ports in `examples/src/bin`.

- `upstream-examples.toml` maps each upstream `.ino` example to the Rust binary that represents it.
- `api-gap-report.md` is generated by `tools/inventory_upstream_examples.py` and records the C++ APIs observed in upstream examples.
- `hardware-verification.md` records ESP-IDF firmware build and on-device validation status.

The target state is that every upstream example has a Rust sample that compiles against the safe `m5unified` crate. Advanced examples may keep third-party Arduino libraries such as ESP8266Audio, A2DP, or AquesTalk as documented feature gates instead of becoming part of the core wrapper API.

Validation:

```bash
python3 tools/check_examples_manifest.py
python3 tools/check_no_sys_in_examples.py
cargo check --workspace
```
